## 1. 프로젝트 한 줄 요약

**DetalSync는 치과와 치기공소 사이의 의뢰 정보, 스캔/STL 파일, 재료·쉐이드·임플란트 정보를 케이스 단위로 정리하고, 기공소의 CAD/CAM 가공 전 휴먼에러를 검증하는 치기공소 운영 자동화 SaaS이다.**

## 3. 현재까지의 핵심 변화

### 초기 가설

초기에는 `지르코니아 디스크 배치 최적화`, `지르코니아 폐기물 재활용`, `네스팅 효율 개선`에서 출발했다.

### 인터뷰 이후 변화

현장 인터뷰를 통해 단순 소재 낭비보다 더 큰 문제는 다음으로 정리되었다.

1. 치과에서 넘어오는 의뢰서, 스캔/STL 파일, 쉐이드, 재료, 임플란트/어버트먼트 정보가 서로 분리되어 있다.
2. 기공소 내부에서 파일 누락, 중복, 수정본/구버전 혼입, 케이스 매칭 오류가 발생한다.
3. 오류가 CAD/CAM 가공 이후에 발견되면 오가공, 재제작, 납기 지연, 치과와의 신뢰 저하로 이어진다.
4. 현재는 숙련자의 암묵지, 수기 체크, 폴더 규칙, 전화/카카오톡/이메일 확인에 의존한다.
5. 따라서 MVP는 완전 자동 네스팅보다 `가공 전 오류 검증 레이어`가 더 현실적이다.

---

## 4. 최종 문제정의 초안

치과 보철 제작 과정은 디지털 장비가 도입되었지만, 실제 운영은 여전히 사람의 확인과 숙련자의 암묵지에 의존한다. 치과에서 전달되는 의뢰서, 스캔/STL 파일, 쉐이드, 재료, 임플란트/어버트먼트 정보가 여러 채널에 분산되어 있어 기공소는 케이스별 정보를 수작업으로 맞춰야 한다. 이 과정에서 파일 누락, 중복, 구버전 사용, 치식 불일치, 재료·부품 매칭 오류가 발생하고, 이는 오가공과 리메이크로 이어진다. DetalSync는 기존 CAD/CAM을 대체하지 않고, 가공 전에 케이스 정보와 파일 상태를 검증하는 운영 레이어로 이 문제를 해결한다.

## 6. 우선 MVP 방향

현재 가장 현실적인 MVP는 다음 세 가지를 조합하는 것이다.

### MVP 1. 의뢰서 OCR 기반 케이스 등록

- 종이/이미지 의뢰서를 OCR로 읽어 케이스를 생성한다.
- OCR 결과는 사용자가 직접 수정할 수 있어야 한다.
- 기존 종이 의뢰서를 당장 없애기보다 `종이+디지털 추적` 하이브리드 방식이 현실적이다.

### MVP 2. 스캔/STL 파일 오류 감지

- 케이스별 파일 누락 감지
- 중복 파일 감지
- 수정본/구버전 혼입 감지
- 케이스 혼입 감지
- 치식 정보 불일치 감지

### MVP 3. 케이스 체크리스트와 오류 로그

- 의뢰서, 파일, 쉐이드, 재료, 임플란트/어버트먼트 정보를 한 화면에서 확인한다.
- 담당자별 작업 상태를 기록한다.
- 오류 로그를 축적해 리메이크 원인과 품질 리포트로 확장한다.

## 7. 핵심 고객 인사이트

### 기공소

- 배치 효율보다 `잘못 깎는 휴먼에러`가 더 직접적인 손실일 수 있다.
- 실사용자는 복잡한 입력 시스템보다 기존 폴더·메일·클라우드 위에서 작동하는 자동 경고를 선호한다.
- 기공소는 추가 입력을 싫어하므로, 자동 감지와 최소 입력이 중요하다.
- 초기 도입 장벽은 기술 자체보다 `검증된 레퍼런스 부족`이다.

### 치과

- 치과는 기공소를 쉽게 바꾸지 않는다.
- 빠른 납기보다 품질 안정성과 리메이크 감소가 더 중요하다.
- 의뢰서 디지털화 자체보다 `결과 품질 개선`과 `재작업 감소`가 구매 논리로 더 강하다.
- 아날로그 인상 기반 치과와 디지털 스캔 기반 치과는 니즈가 다르다.

## 10. 다른 AI에게 요청할 때 주의할 점

- DetalSync를 단순 `치과 의뢰 플랫폼`으로 축소하지 말 것.
- DetalSync를 단순 `네스팅 최적화 프로그램`으로 설명하지 말 것.
- 핵심은 `가공 전 오류 검증`, `기공소 운영 레이어`, `리메이크/오가공 감소`, `치과와 기공소 신뢰 회복`이다.
- 근거 없는 시장 규모, 수치, 논문 인용을 만들지 말 것.
- 불확실한 내용은 반드시 `추가 검증 필요`로 표시할 것.
